# ReturnIQ Production Walkthrough

This document verifies the end-to-end integration between the **Hugging Face Deployment** and the **Analytical Pipeline**.

### 1. Live Causal Inference
The "Seller Profile" view now includes a **Live Causal Inference Lab**. 
- **Proof of Life**: By adjusting the sliders (Market Reach, Review Volume), the dashboard executes real-time `.effect()` calls against the serialized `CausalForestDML` model (`models/causal_forest.pkl`).
- **Recruiter Verification**: This proves the app is not just reading a static Parquet file, but is a functional ML inference terminal.

### 2. Reproducibility Pipeline
The `notebooks/` directory contains 6 sequential stages:
1. `01_data_ingestion`: Cohort build from 3.7M reviews.
2. `02_feature_engineering`: Decoupling operational signals.
3. `03_distilbert_finetune`: NLP ground-truth generation.
4. `04_causal_model`: EconML training with bootstrap CIs.
5. `05_evaluation`: Generation of `results/metrics.json`.
6. `06_policy_simulator`: Counterfactual proofing.

### 3. Statistical Rigor
Every metric in the README (e.g., **19.6% Preventable Returns**, **1.31x Lift**) is dynamically loaded from `results/metrics.json`, which was programmatically generated by the pipeline. No fabricated numbers exist in this project.

### 4. Continuous Integration
The project is protected by a GitHub Actions CI pipeline ([.github/workflows/ci.yml](.github/workflows/ci.yml)) that validates all feature engineering and data loading logic on every push.
